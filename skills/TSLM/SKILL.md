# 流量预测

基于时序大模型的洪水预测，支持单变量和协变量模式。

## 使用场景

✅ **USE when:**
- 用户要求预测流量
- 用户直接提供了历史流量数据（如"流量为 150、160、170、180、165 m³/s"）
- 需要预测未来流量值

❌ **DON'T use when:**
- 用户需要先查看数据文件结构（先用 xlsx 技能的 preview_data.py 预览）
- 要评估模型历史精度（用 validation 技能）
- 没有历史数据

## 可执行脚本

使用 `run_script` 工具执行以下脚本：

### flood_prediction.py流量预测

**必要条件：**
- 历史数据至少需要 5 个时间点
- 时间格式为 `YYYY-MM-DD HH:MM:SS`
- 协变量数据长度须与历史数据一致

**用法：**
```bash
python flood_prediction.py --times '["2025-01-01 00:00:00", ...]' --flows '[120.5, 125.3, ...]' [options]
```

**参数：**
| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| --times | ✅ | - | 历史时间序列（JSON数组） |
| --flows | ✅ | - | 历史流量序列（JSON数组） |
| --predict_steps | ❌ | 8 | 预测步数 |
| --mode | ❌ | univariate | 预测模式 |
| --past_covariates | ❌ | - | 历史协变量（JSON对象） |
| --future_covariates | ❌ | - | 未来协变量（JSON对象） |
| --output_file | ❌ | - | 把结构化预测结果保存为 JSON 文件，父目录会自动创建 |

**预测模式说明：**
| 模式 | 说明 | 适用场景 |
|------|------|---------|
| univariate | 单变量预测 | 仅用历史流量预测 |
| past_covariates | 含过去协变量 | 加入历史降雨、上游流量等 |
| future_covariates | 含未来协变量 | 加入气象预报等未来已知信息 |

**输出形式：**
- 返回预测结果字符串，包含：预测模式、历史数据点数、预测步数、推断时间间隔、预测结果（时间、中位数、置信区间）
- 如果提供 `--output_file`，会把结构化预测结果保存为 JSON 文件，同时 stdout 仍返回人类可读结果

## 大时序输入优先策略

当 `--times`、`--flows`、`--past_covariates` 或 `--future_covariates` 内容较长时：

1. 不要优先在 `run_script` 的 `args` 中直接内联超长 JSON 数组或对象
2. 应优先先整理到本地文件，再由脚本或中间步骤读取后执行预测
3. 若数据来自表格、CSV、Excel 或长文本，优先先做文件整理，不要手工拼接超长参数
4. 只有在历史点数较少、参数较短时，才适合直接内联 `--times` 和 `--flows`

**调用示例：**

基础预测：
```
run_script(
    skill_name='prediction',
    script_name='flood_prediction.py',
    args='["--times", "[\"2025-01-01 00:00:00\", \"2025-01-01 01:00:00\", \"2025-01-01 02:00:00\", \"2025-01-01 03:00:00\", \"2025-01-01 04:00:00\"]", "--flows", "[120.5, 125.3, 130.2, 128.7, 132.1]", "--predict_steps", "8", "--output_file", "chronos_result.json"]'
)
```

含协变量预测：
```
run_script(
    skill_name='prediction',
    script_name='flood_prediction.py',
    args='["--times", "[\"2025-01-01 00:00:00\", ...]", "--flows", "[120.5, 125.3, ...]", "--mode", "past_covariates", "--past_covariates", "{\"rainfall\": [5.2, 3.1, ...]}"]'
)
```

**输出示例：**
```
洪水流量预测结果
预测模式      : 单变量
历史数据点数  : 5
预测步数      : 8
推断时间间隔  : 60 分钟

预测结果（中位数  [10% ~ 90% 置信区间]）：
   1. 2025-01-02 00:00:00      125.50 m³/s  [110.20 ~ 140.80]
   2. 2025-01-02 01:00:00      128.30 m³/s  [112.50 ~ 145.10]
   ...
```

## 数据输入方式

### 方式1：用户直接提供流量数值

当用户说"现在吴堡站流量为 150、160、170、180、165 m³/s（每隔30分钟），预测未来4小时"时：

1. **模型自动解析**用户输入，提取：
   - 流量值：`[150, 160, 170, 180, 165]`
   - 时间间隔：30分钟
   - 预测时长：4小时 → 8步预测

2. **模型自动生成时间序列**，从当前时间开始，每隔30分钟：
   ```json
   ["2025-03-25 10:00:00", "2025-03-25 10:30:00", "2025-03-25 11:00:00", "2025-03-25 11:30:00", "2025-03-25 12:00:00"]
   ```

3. **直接调用** `flood_prediction.py` 进行预测

### 方式2：用户提供数据文件

当用户提供数据文件时，应使用 `validation` 技能进行模型验证，或使用 `xlsx` 技能的 `preview_data.py` 预览文件结构。

## 工作流程

```
用户输入: "流量为 150、160、170、180、165 m³/s（每隔30分钟），预测未来4小时"
    ↓
模型解析: times=[...], flows=[150, 160, 170, 180, 165], predict_steps=8
    ↓
flood_prediction.py --times --flows --predict_steps
    ↓
预测结果
```

## 注意事项

1. 历史数据至少需要 5 个时间点
2. 时间格式为 `YYYY-MM-DD HH:MM:SS`
3. 如果有协变量数据，会自动切换到 `past_covariates` 模式
4. 协变量数据长度须与历史数据一致
5. **模型会自动解析用户输入，无需中间数据转换步骤**
